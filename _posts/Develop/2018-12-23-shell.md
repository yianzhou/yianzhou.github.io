You can find out what shell you are using: `echo $SHELL`

`$PATH`: contains a colon separated list of directories indicating where the Unix operating system expects executable programs to be found

`$EDITOR`: default editor

`PWD`: Specifies the current working directory.

`USER`: the current user

`$RANDOM`: Contains a random integer which changes each time the variable is referenced.

If you are using SH or some variant you can change environment variables temporarily from the command line using the following syntax: `export NAME=value`.

If you are using SH or some variant the script executed at login will be .profile. Any settings in this/these file(s) will effect the user whom owns them.

If you would like to change an environment variable permanently you should edit the file .profile.

Manual page: `man command`

We can specify the shell we want to interpret in the first line: `#!/bin/bash`

We use `>` to redirect stdout to a file.

Surrounding anything within single quotes causes it to be treated as literal text that is it will be passed on exactly as intended.
