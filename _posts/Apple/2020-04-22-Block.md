# block 的声明

![image](/assets/images/blocks_2x.png)

As a local variable:
```objc
returnType (^blockName)(parameterTypes) = ^returnType(parameters) {...};
```

As a property:
```objc
@property (nonatomic, copy, nullability) returnType (^blockName)(parameterTypes);
```

As a method parameter:
```objc
- (void)someMethodThatTakesABlock:(returnType (^nullability)(parameterTypes))blockName;
```

As an argument to a method call:
```objc
[someObject someMethodThatTakesABlock:^returnType (parameters) {...}];
```

As a parameter to a C function:
```c
void SomeFunctionThatTakesABlock(returnType (^blockName)(parameterTypes));
```

As a typedef:
```objc
typedef returnType (^TypeName)(parameterTypes);
TypeName blockName = ^returnType(parameters) {...};
```

# block 的本质

block 是封装了函数调用以及函数调用环境的 OC 对象。

# block 的底层

想探究 block 的底层，可以写一个 `main.m` 文件：

```c
int main(int argc, const char * argv[]) {
    int a = 1;
    __block int b = 2;
    void (^block)(void) = ^{
        printf("%d", a);
        b++;
    };
    block();
    return 0;
}
```

然后用以下指令将其转化成 C++ 代码：

```
xcrun -sdk iphoneos clang -arch arm64 -rewrite-objc main.m
```

打开 main.cpp，找到其中 block 实现的部分：

```cpp
struct __main_block_impl_0 {
  struct __block_impl impl;
  struct __main_block_desc_0* Desc;
  // 结构体内部存放了 a（值）和 b（地址）
  int a;
  __Block_byref_b_0 *b; // by ref
  __main_block_impl_0(void *fp, struct __main_block_desc_0 *desc, int _a, __Block_byref_b_0 *_b, int flags=0) : a(_a), b(_b->__forwarding) {
    impl.isa = &_NSConcreteStackBlock;
    impl.Flags = flags;
    impl.FuncPtr = fp; // 指向 block 函数的指针
    Desc = desc;
  }
};
// block 的实现变成了函数
static void __main_block_func_0(struct __main_block_impl_0 *__cself) {
    __Block_byref_b_0 *b = __cself->b; // bound by ref 引用绑定
    int a = __cself->a; // bound by copy 值绑定
    printf("%d", a);
    (b->__forwarding->b)++;
}
int main(int argc, const char * argv[]) {
    int a = 1; // 原封不动
    __attribute__((__blocks__(byref))) __Block_byref_b_0 b = {(void*)0,(__Block_byref_b_0 *)&b, 0, sizeof(__Block_byref_b_0), 2}; // 经过编译器优化，变成结构体，传入变量 b 的地址

    // __main_block_impl_0 是 block 底层的结构体
    // 调用结构体的构造函数，a传值，b传的是地址
    void (*block)(void) = ((void (*)())&__main_block_impl_0((void *)__main_block_func_0, &__main_block_desc_0_DATA, a, (__Block_byref_b_0 *)&b, 570425344));
    ((void (*)(__block_impl *))((__block_impl *)block)->FuncPtr)((__block_impl *)block);
    return 0;
}
```

# \_\_block 修饰符

```objc
NSMutableArray *array = [NSMutableArray array];
void(^block)(void) = ^{
    [array addObject:@123];
};
block();
```

这里 对 array 只是使用，而不是赋值，所以不需要 `__block` 进行修饰；

```objc
NSMutableArray *array = nil;
void(^block)(void) = ^{
    array = [NSMutableArray array];
};
block();
```

这里就需要在 array 的声明处添加 `__block` 修饰符，否则编译器会报错。

# block 根据内存分布情况可分为三类

在 Objective-C 语言中，一共有 3 种类型的 block。

第一种：`_NSConcreteGlobalBlock` 全局的静态 block。当 block 没有捕获任何外部的变量，它的整块内存空间在编译期就已经清楚了，这时它属于一个全局 block，加载在内存的 data section 区。

```c
// main.m
int main(int argc, const char * argv[]) {
    int a = 10;
    void (^block)(int a) = ^(int a){
        NSLog(@"a = %d",a);
    };
    NSLog(@"%@", block); // <__NSGlobalBlock__: 0x100001028>
    block(a);
    return 0;
}
```

第二种：\_NSConcreteStackBlock 保存在栈中的 block，当 block 捕获外部变量时，就会放到栈区。

```c
// main.m
int main(int argc, const char * argv[]) {
    int a = 10;
    void (^block)(void) =  ^{
        NSLog(@"a = %d",a);
    };
    block();
    NSLog(@"%@", [block class]); // __NSStackBlock__
    return 0;
}
```

第三种：\_NSConcreteMallocBlock 保存在堆中的 block

```c
// main.m
int main(int argc, const char * argv[]) {
    int a = 10;
    void (^block)(void) =  ^{
        NSLog(@"a = %d",a);
    };
    block();
    NSLog(@"%@", [block class]); // __NSMallocBlock__
    return 0;
}
```

二、三代码完全一样，区别是二是 MRC，三是 ARC。在 ARC 环境下，编译器会根据情况自动将栈上的 block 复制到堆上。

# block 引起循环引用的三种解决方式

## 1. weak-strong dance

```
self -> block -> strong -> weak -> self
```

Objective-C:

```objc
    __weak typeof(self) weakSelf = self;
    self.block = ^{
        __strong typeof(weakSelf) strongSelf = weakSelf; // weak-strong dance
        dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(2 * NSEC_PER_SEC)), dispatch_get_main_queue(), ^{
            NSLog(@"%@", strongSelf.name);
        });
    };
    self.block();
```

Swift:

```swift
    self.block = { [weak self] in
        guard let strongSelf = self else { return } // weak-strong dance
        DispatchQueue.main.asyncAfter(deadline: .now() + 2.0) {
            print(strongSelf.name)
        }
    }
    self.block()
```

## 2. 临时变量

```objc
    __block ViewController* vc = self;
    self.block = ^{
        dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(2 * NSEC_PER_SEC)), dispatch_get_main_queue(), ^{
            NSLog(@"%@", vc.name);
            vc = nil;
        });
    };
    self.block();
```

## 3. block 传参

```objc
    self.block = ^(ViewController *vc){
        dispatch_after(dispatch_time(DISPATCH_TIME_NOW, (int64_t)(2 * NSEC_PER_SEC)), dispatch_get_main_queue(), ^{
            NSLog(@"%@", vc.name);
        });
    };
    self.block();
```

# block 应用举例

```objc
- (void)viewDidLoad {
    self.button = [[UIButton alloc]init];
    __weak typeof(self) weakSelf = self;
    self.button.onClick = ^(UIButton *button){
        if (weakSelf == nil) {
            return;
        }
        // 在 block 里面使用实例变量，会强引用，编译器会警告
        [_manager reloadData];
        // 使用 weak 指针调用实例变量，编译器会报错
        [weakSelf->_manager reloadData];
        // 需要转成 strong 指针
        __strong typeof(self) strongSelf = weakSelf;
        [strongSelf->_manager reloadData];
    };
}
```
