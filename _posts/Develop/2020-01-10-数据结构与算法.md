# 2. 算法分析

算法分析关心的一是考虑算法在解决问题时要占用的空间或内存。解决方案所需的空间总量一般由问题实例本身决定，但算法往往也会有特定的空间需求，后文会详细介绍。

另一个是算法执行所需的时间。

考虑解决“求前 n 个整数和”问题的算法：

```python
def sumOfN(n):
    sum = 0
    for i in range(1, n+1):
        sum = sum + i
    return sum
```

在 sumOfN 函数中，运行次数是 1(theSum = 0 )加上 (theSum = theSum + i) 的运行次数。可以将其定义成函数 T，令 T(n) = 1 + n。参数 n 常被称作问题规模，可以将函数解读为“当问题规模为 n 时，解决问题所需的时间是 T(n)，即需要 n 步”。

数量级 (order of magnitude) 描述的是，当 n 增长时，T(n) 增长最快的部分，常被称作大 O 记法，记作 O(f(n))，常见的大 O 函数[参考这里](https://io.yianzhou.com/lego/data-structure-and-algorithms/1.-fundamental/1.4-analysis-of-algorithms)。

算法的性能有时不仅依赖于问题规模，还依赖于数据值。要用最坏情况、最好情况、普通情况来描述性能，以免被某个特例误导。

## 例：[anagram 问题(LeetCode-242)](https://leetcode.com/problems/valid-anagram/submissions/)的四种解法

1. 暴力穷举法：用 s1 中的字符生成所有的可能，看是否跟 s2 匹配，复杂度为 n!，当 n=20 时，有 2432902008176640000 种排列，这是一个比 2^n 要快得多的增长！

2. 双层循环：s1 的字符逐个与 s2 比较，复杂度为 O(n^2)。

3. 排序法：s1 和 s2 分别排序，再逐个比较。复杂度为 O(nlogn)。

4. 计数法：两个异序词必有相同数目的 a,b,c,... 使用计数器统计每个字符出现的次数，复杂度为 O(n)。

## Python 数据结构与性能分析

There are three basic sequence types: **lists**, **tuples**, and **range** objects.

**Strings** are immutable sequences of Unicode code points.

The core built-in types for manipulating binary data are **bytes** and **bytearray**. They are supported by **memoryview** which uses the buffer protocol to access the memory of other binary objects without needing to make a copy.

A **set** object is an unordered collection of distinct hashable objects.

A mapping object maps hashable values to arbitrary objects. Mappings are mutable objects. There is currently only one standard mapping type, the **dictionary**.

Python 是一门变化中的语言，内部实现一直会有更新。可以在官网上找到 Python 数据结构性能的最新信息。

# 3. 基本数据结构

## 线性数据结构

元素被添加后，它与前后元素的相对位置保持不变，例如栈、队列、双端队列、列表。

### 栈

栈的实现：用 python 的原生列表来实现栈，将列表的末尾作为栈的顶端；也可以选择将列表的头部作为栈的顶端，那就是另一种实现。两种实现在性能上会有很大差异。

栈的应用：

- 叠盘子、浏览器/ViewController 的导航；
- [匹配括号(LeetCode-20)](https://leetcode.com/problems/valid-parentheses/)；
- 将十进制数转换为任意进制数；
- 自主实现一个[计算器(LeetCode-224)](https://leetcode.com/problems/basic-calculator/)；

### 队列

队列的实现：用 python 的原生列表来实现队列，假设队列的尾部在列表的位置 0 处。意味着添加操作 `insert(0, item)` 的时间复杂度是 O(n)；移除操作 `pop()` 的时间复杂度是 O(1)

队列的应用：

- 时序模拟，如打印任务排队；

### 双端队列

双端队列的实现：列表的位置 0 处是队列的后端。在前段进行添加和移除的时间复杂度是 O(1)；在后端则是 O(n)

双端队列的应用：

- 回文检测器 (Palindrome, [LeetCode-125](https://leetcode.com/problems/valid-palindrome/))

### 链表

python 的原生列表不是通过链表实现的，是通过数组实现的。

无序列表`UnorderedList`需要维持元素之间的相对位置，但是并不需要在连续的内存空间中维护。我们可以用链表 (LinkedList) 实现无序列表，每一个元素维护了下一个元素的位置信息。

有序列表`OrderedList`中，假设元素之间是 Comparable，元素的相对位置以升序或降序排列。

# 4. 递归

递归是解决问题的一种方法，它将问题不断地分解成更小的子问题，直到子问题可以用普通的方法解决。递归会使用一个不断调用自己的函数。

递归三原则：1. 退出条件；2. 不断向退出条件靠近；3. 递归调用自己

应用举例：将十进制数转换为任意进制数；汉诺塔；turtle 模块绘制分形树/sierpinski 三角形；走出迷宫

许多计算机程序被用于优化某些值。解决优化问题的一个策略是**动态规划** (Dynamic Programming)。

优化问题的一个经典例子，是在[找零时使用最少的硬币(LeetCode-322)](https://leetcode.com/problems/coin-change/)。先使用面值最大的货币、再使用第二大的货币……这种方法叫做**贪婪算法**。但如果硬币的面额除了 1、5、10、25 以外还有其它面额，则贪婪算法不一定能得到最优解。要想必定得到最优解，需使用动态规划。

在解决找零问题时，动态规划算法会从 1 分开始，系统化地一直计算到所需地找零金额，以保证，每一步都已经知道任何小于当前值所需的最少硬币数。比如找零金额是 11 分，则从 1 分开始计算，计算 6 分前，找零 1-5 分所需的最少硬币数都已经记录下来。动态规划方法不一定是递归，在这个例子中，我们的动态规划方法并不是递归。

# 5. 搜索与排序

## 搜索

无序列表的顺序搜索。

有序列表的二分搜索：分治策略 (Divide and Conquer)，将问题分解成更小的问题，解决小问题，然后逐步整合结果。

## Hash table

Hash table 是一个元素集合，其中的元素以便于查找的方式存储。Hast table 中的每个位置称为 slot。

```
load factor = n / k
```

n 是元素个数，k 是 hash table 的 size。常常建议散列表的大小为素数。

当 hash function 把两个元素放入同一个 slot 时，就发生了冲突 (ccollision)。Rehash 指发生冲突后，寻找另一个 slot 的过程。处理冲突的方式包括：线性探测、平方探测、链接法等。

## 排序

冒泡排序、选择排序、插入排序：O(n^2)、希尔排序：O(n^3/2)

归并排序（需要额外内存空间）、快速排序（原地）：O(logN)

排序算法稳定性定义：排序前后两个相等的数相对位置不变，则算法稳定。归并排序是稳定的排序算法；快速排序不是稳定的排序算法。

稳定性的好处：从一个键上排序，然后再从另一个键上排序，第一个键排序的结果可以为第二个键排序所用。例如：排序的内容是一组原本按照价格高低排序的对象，如今需要按照销量高低排序，使用稳定性算法，可以使得相同销量的对象，依旧保持着价格高低的排序展现，只有销量不同的才会重新排序。
