# lldb 命令

`$po bt all` 打印全部堆栈信息（print object, backtrace）

`(lldb) print <expr>` : 打印需要查看的变量，简写 p，你会发现输出的信息中带有$1、$2 的字样。实际上，lldb 的每次查询结果会保存在一些持续变量中，(\$[0-9]+)，之后可以直接使用、修改这些值。

# 崩溃符号化

一、导出崩溃日志，设置 -> 隐私 -> 分析与改进 -> 分析数据，根据崩溃的发生时间和 App 名字，找到 ips 文件，点击查看，然后通过导航栏的分享按钮发送出来。

二、Xcode 中必须要有对应版本系统的 iOS DeviceSupport 文件，否则符号化会失败。打开 DeviceSupport 文件夹：

`open /Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/DeviceSupport`

下载 DeviceSupport：<https://github.com/iGhibli/iOS-DeviceSupport>

三、找到发生崩溃的测试包对应的 dSYMs 文件。将崩溃日志和 dSYMs 文件放到同一个目录下，然后采用 symbolicatecrash 命令进行符号化。symbolicatecrash 是 Xcode 自带的符号化工具，但是不在系统的 PATH 内，因此运行该命令需要修改 PATH 变量，在终端执行如下命令：

```
export DEVELOPER_DIR="/Applications/Xcode.app/Contents/Developer"
export PATH=$PATH:/Applications/Xcode.app/Contents/SharedFrameworks/DVTFoundation.framework/Versions/A/Resources/
```

执行命令：`symbolicatecrash xxx.crash -o yyy.crash`

xcconfig 的配置

1. 新建 Debug.xcconfig 和 Release.xcconfig 文件。
2. Open your project in Xcode, and select the project file in the Xcode Navigator. Select project from the Select a project or target dropdown.
3. 配置 Configurations 为我们新建的 xcconfig。
