# 算法分析

算法分析关心的，一是考虑算法在解决问题时要占用的空间；二是算法执行所需的时间。

当问题规模为 n 时，解决问题所需的时间是 `T(n)`。数量级 (order of magnitude) 描述的是，当 n 增长时，`T(n)` 增长最快的部分，常被称作大 O 记法，记作 `O(T(n))`，常见的大 O 函数[参考这里](https://io.yianzhou.com/lego/data-structure-and-algorithms/1.-fundamental/1.4-analysis-of-algorithms)。

算法的性能有时不仅依赖于问题规模，还依赖于数据值。要用最坏情况、最好情况、普通情况来描述性能，以免被某个特例误导。

## 例：[anagram 问题(LeetCode-242)](https://leetcode.com/problems/valid-anagram/submissions/)的四种解法

1. 暴力穷举法：用 s1 中的字符生成所有的可能，看是否跟 s2 匹配，复杂度为 n!，当 n=20 时，有 2432902008176640000 种排列，这是一个比 2^n 还要快得多的增长！

2. 双层循环：s1 的字符逐个与 s2 比较，复杂度为 O(n^2)。

3. 排序法：s1 和 s2 分别排序，再逐个比较。复杂度为 O(nlogn)。

4. 计数法：两个异序词必有相同数目的 a,b,c,... 使用计数器统计每个字符出现的次数，复杂度为 O(n)。

# 基本数据结构

**数据结构的存储方式只有两种：数组（顺序存储）和链表（链式存储）。**

散列表、栈、队列、堆、树、图等等，都是在链表或者数组上的特殊操作，属于“上层建筑”。

线性数据结构：元素被添加后，它与前后元素的相对位置保持不变，例如栈、队列、双端队列、列表。这类结构，既可以使用链表也可以使用数组实现。用数组实现，就要处理扩容缩容的问题；用链表实现，没有这个问题，但需要更多的内存空间存储节点指针。

「图」的两种表示方法，邻接表就是链表，邻接矩阵就是二维数组。邻接矩阵判断连通性迅速，并可以进行矩阵运算解决一些问题，但是如果图比较稀疏的话很耗费空间。邻接表比较节省空间，但是很多操作的效率上肯定比不过邻接矩阵。

「散列表」就是通过散列函数把键映射到一个大数组里。而且对于解决散列冲突的方法，拉链法需要链表特性，操作简单，但需要额外的空间存储指针；线性探查法就需要数组特性，以便连续寻址，不需要指针的存储空间，但操作稍微复杂些。

「树」，用数组实现就是「堆」，因为「堆」是一个完全二叉树，用数组存储不需要节点指针，操作也比较简单；用链表实现就是常见的「树」，衍生出各种巧妙的设计，比如二叉搜索树、AVL 树、红黑树等等，以应对不同的问题。

栈的应用：

- 叠盘子、浏览器/ViewController 的导航；
- [匹配括号(LeetCode-20)](https://leetcode.com/problems/valid-parentheses/)；
- 将十进制数转换为任意进制数；
- 自主实现一个[计算器(LeetCode-224)](https://leetcode.com/problems/basic-calculator/)；

队列的应用：

- 时序模拟，如打印任务排队

双端队列的应用：

- 回文检测器 (Palindrome, [LeetCode-125](https://leetcode.com/problems/valid-palindrome/))

# 递归

递归是解决问题的一种方法，它将问题不断地分解成更小的子问题，直到子问题可以用普通的方法解决。递归会使用一个不断调用自己的函数。

递归三原则：1. 退出条件；2. 不断向退出条件靠近；3. 递归调用自己

应用举例：将十进制数转换为任意进制数；汉诺塔；turtle 模块绘制分形树/sierpinski 三角形；走出迷宫

许多计算机程序被用于优化某些值。解决优化问题的一个策略是**动态规划** (Dynamic Programming)。

优化问题的一个经典例子，是在[找零时使用最少的硬币(LeetCode-322)](https://leetcode.com/problems/coin-change/)。先使用面值最大的货币、再使用第二大的货币……这种方法叫做**贪婪算法**。但如果硬币的面额除了 1、5、10、25 以外还有其它面额，则贪婪算法不一定能得到最优解。要想必定得到最优解，需使用动态规划。

在解决找零问题时，动态规划算法会从 1 分开始，系统化地一直计算到所需地找零金额，以保证，每一步都已经知道任何小于当前值所需的最少硬币数。比如找零金额是 11 分，则从 1 分开始计算，计算 6 分前，找零 1-5 分所需的最少硬币数都已经记录下来。

动态规划方法不一定是递归。

# 搜索与排序

## 搜索

无序列表的顺序搜索。

有序列表的二分搜索：分治策略 (Divide and Conquer)，将问题分解成更小的问题，解决小问题，然后逐步整合结果。

## Hash table

Hash table 是一个元素集合，其中的元素以便于查找的方式存储。Hast table 中的每个位置称为 slot。

```
load factor = n / k
```

n 是元素个数，k 是 hash table 的 size。常常建议散列表的大小为素数。

当 hash function 把两个元素放入同一个 slot 时，就发生了冲突 (ccollision)。Rehash 指发生冲突后，寻找另一个 slot 的过程。处理冲突的方式包括线性探测和链接法。

- 线性探测：顺序遍历哈希表，直到找到一个空槽。
- 链接法：让每个槽有一个指向元素集合（链表）的引用。

## 排序

冒泡排序、选择排序、插入排序：O(n^2)、希尔排序：O(n^3/2)

归并排序：将列表一分为二，排序左半边，再排序右半边，最后合并，递归进行。

快速排序：与归并排序的等分操作不同，先选取基准值并分区，对左半区、右半区递归排序。

归并排序和快速排序的时间复杂度都是 O(logN)，不同的是归并排序需要额外的内存空间，快排是原地。

排序算法稳定性定义：排序前后两个相等的数相对位置不变，则算法稳定。归并排序是稳定的排序算法；快速排序不是稳定的排序算法。

稳定性的好处：从一个键上排序，然后再从另一个键上排序，第一个键排序的结果可以为第二个键排序所用。例如：排序的内容是一组原本按照价格高低排序的对象，如今需要按照销量高低排序，使用稳定性算法，可以使得相同销量的对象，依旧保持着价格高低的排序展现，只有销量不同的才会重新排序。

# 树

树的应用：生物学分类、文件目录、网页 HTML 代码的标签嵌套

树的定义：

定义一：树由节点及连接节点的边构成。树有以下属性：有一个根节点；除根节点外，其余节点都与唯一的父节点相连；从根节点到其它每个节点有且仅有一条路径；如果每个节点最多有两个子节点，我们就称这样的树是**二叉树**。

定义二：一棵树要么为空，要么由一个根节点和零或多颗子树构成。每棵子树的根节点通过一条边连接到父树的根节点。

我们根据定义二，用 Python 代码实现二叉树。

二叉树的应用：解析句子结构，或数学表达式。

# 图

有两种非常著名的图的抽象数据类型：邻接矩阵和邻接表。本节会用邻接表来实现图。

邻接矩阵的优点是简单，缺点是大部分现实问题都是很“稀疏”的，只有当图中的每一个点与其他的所有点都连接时，邻接矩阵才会被填满。

在邻接表实现中，我们为图对象的所有顶点保存一个主列表，同时为每一个顶点对象都维护一个列表，其中记录了与它相连的顶点。

有向无环图（DAG, Directed acyclic graph）

广度优先算法：二叉树的层序遍历、词梯问题

深度优先算法：骑士周游问题
