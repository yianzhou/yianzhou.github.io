# 结构

> objc.h

```objc
#if !OBJC_TYPES_DEFINED
/// An opaque type that represents an Objective-C class.
typedef struct objc_class *Class;

/// Represents an instance of a class.
struct objc_object {
    Class _Nonnull isa  OBJC_ISA_AVAILABILITY;
};

/// A pointer to an instance of a class.
typedef struct objc_object *id;
#endif
```

> runtime.h

```objc
struct objc_class {
    Class _Nonnull isa  OBJC_ISA_AVAILABILITY;

#if !__OBJC2__
    Class _Nullable super_class    OBJC2_UNAVAILABLE;
    const char * _Nonnull name     OBJC2_UNAVAILABLE;
    long version                   OBJC2_UNAVAILABLE;
    long info                      OBJC2_UNAVAILABLE;
    long instance_size             OBJC2_UNAVAILABLE;
    struct objc_ivar_list * _Nullable ivars                  OBJC2_UNAVAILABLE;
    struct objc_method_list * _Nullable * _Nullable methodLists    OBJC2_UNAVAILABLE;
    struct objc_cache * _Nonnull cache                       OBJC2_UNAVAILABLE;
    struct objc_protocol_list * _Nullable protocols          OBJC2_UNAVAILABLE;
#endif

} OBJC2_UNAVAILABLE;
```

# 例子

objc 的源码下载：<https://github.com/opensource-apple/objc4>

如何跳进汇编代码：Xcode 菜单栏 - Debug - Debug Workflow - Always Show Disassembly

```
Demo* demo = [[Demo alloc] init];
[demo test];
```

在内存中，`demo` 对象是 `objc_object` 结构体，`Demo` 类是 `objc_class` 结构体。

方法的调用实际上是发送消息：`objc_msgSend(demo, @selector(test), params)`

通过 isa 指针找到类对象(objc_class)，里面有 `struct objc_method_list` 和 `struct objc_cache`。

根据 selector 的名字，在类对象的缓存列表中查找（本质是字符串匹配），如果没有命中，则在方法列表中查找，找到后缓存下来，并
跳转到方法的实现；下次发送同一消息时，命中缓存列表，直接跳转到方法的实现。

如果没有找到，通过 `super_class` 指针往父类里查找，一直到 `NSObject` 都没有找到的话，则启动消息转发机制。

纯 Swift 类，在编译期即通过 V-Table 确定了函数的调用，直接从虚函数表通过索引取出函数的内存地址来调用，不具备运行时消息转
发机制。

# NSObject

NSObject 对象在底层的结构体是 `objc_object`，它的内容只有`isa`指针，指针在 64 位系统中占用内存空间是 8 字节；对象在分配内存空间时，会进行内存对齐，所以在 iOS 中，NSObject 所占内存是 16 字节。

ObjC 类中的属性、方法还有遵循的协议等信息都保存在 `class_rw_t` 中；`class_ro_t` 中存储了当前类在编译期就已经确定的属性、方法以及遵循的协议。

`​NSObject` is the root class of most Objective-C class hierarchies, from which subclasses inherit a basic interface to
the runtime system and the ability to behave as Objective-C objects.

An object that conforms to NSObject protocol can be considered a first-class object. Such an object can be asked about
its:

- Class, and the place of its class in the inheritance hierarchy;
- Conformance to protocols;
- Ability to respond to a particular message.

The Cocoa root class `NSObject` adopts this protocol. This protocol is imported into Swift with the name
`NSObjectProtocol`.

`-(BOOL)isMemberOfClass:(Class)aClass;` Returns a Boolean value that indicates whether the receiver is an instance of a
given class.

`-(BOOL)isKindOfClass:(Class)aClass;` Returns a Boolean value that indicates whether the receiver is an instance of
given class or an instance of any class that inherits from that class.

`+(instancetype)alloc;` Returns a new instance of the receiving class.

`-(instancetype)init` Implemented by subclasses to initialize a new object (the receiver) immediately after memory for
it has been allocated.

`+(instancetype)new;` Allocates a new instance of the receiving class, sends it an init message, and returns the
initialized object.

`+ (void)load;` Invoked whenever a class or category is added to the Objective-C runtime; implement this method to perform class-specific behavior upon loading.

`+ (void)initialize;` Initializes the class before it receives its first message. The runtime sends initialize to each class in a program just before the class, or any class that inherits from it, is sent its first message from within the program.

# Category

> [深入理解 Objective-C：Category](https://tech.meituan.com/2015/03/03/diveintocategory.html)

objc 的 category 实际上是 `category_t` 结构体：

```c
struct category_t {
    const char *name;
    classref_t cls;
    struct method_list_t *instanceMethods;
    struct method_list_t *classMethods;
    struct protocol_list_t *protocols;
    struct property_list_t *instanceProperties;
};
```

可以看出，category 可以添加实例方法，类方法，甚至可以实现协议，添加属性；但无法添加实例变量。

category 是如何被加载的：先创建主类，然后将所有 category 附加到主类。

在类的 `+load` 方法调用的时候，我们可以调用 category 中声明的方法么？——可以。因为附加 category 到类的工作会先于 `+load` 方法的执行。

两个 category 的 load 方法的执行顺序：`+load` 的执行顺序是先类，后 category，而 category 的 `+load` 执行顺序是根据编译顺序决定的。在 Compile Sources 中改变 Category 的顺序，会影响执行顺序。

当父类和子类都实现 load 函数时，父类的 load 方法执行顺序要优先于子类。当子类未实现 load 方法时，不会调用父类 load 方法。

两个 category 的同名方法：category 是逐个附加到主类上的，新加的方法会排在列表前面，查找方法时也会优先被匹配到。（objc 源码中的 `remethodizeClass` 方法）

[class extensions are often referred to as anonymous categories](https://developer.apple.com/library/archive/documentation/Cocoa/Conceptual/ProgrammingWithObjectiveC/CustomizingExistingClasses/CustomizingExistingClasses.html#//apple_ref/doc/uid/TP40011210-CH6-SW3), class extension 和 category 的区别：extension 是你有源代码时才能添加，且能添加成员变量。无法为 `NSObject` 添加 class extension。

# weak

Runtime 维护了一个 weak 表，用于存储指向某个对象的所有 weak 指针。weak 表其实是一个 hash 表，Key 是所指对象的地址，Value 是 weak 指针的地址数组。

当对象不再被强引用、需要销毁时，通过表找到所有的弱引用指针，置为空。

SideTable 结构体存放引用计数表和 weak 表。

```c
struct SideTable {
    // ...
    RefcountMap refcnts;
    weak_table_t weak_table;
}
```
